# PresenterOverlay (Суфлёр) — Python

Кроссплатформенная версия «Суфлёра»: полупрозрачный плавающий оверлей, который
распознаёт твою речь **локально через Whisper** и отправляет текст в **DeepSeek**.
DeepSeek по контексту исправляет ошибки распознавания и даёт короткий готовый
ответ, который можно сразу произнести. Окно по возможности **исключается из
захвата экрана** — при демонстрации презентации зрители его не видят.

Работает на **Windows, macOS и Linux** — нужен только Python 3.9+.

## Почему так
- **Whisper локально** (`faster-whisper`) — распознаёт офлайн, без облака и без ключей.
- **DeepSeek как корректор** — обычный распознаватель может перепутать похожие по
  звучанию слова; модель видит весь контекст и восстанавливает смысл.
- **Слова сразу в поле ввода** — каждое распознанное слово тут же появляется в
  поле, так что видно всё, что было проговорено, и текст можно поправить руками.

## Быстрый запуск

### Windows
Двойной клик по `run.bat` (или в терминале):
```bat
run.bat
```

### macOS / Linux
```bash
./run.sh
```

Скрипты сами создают виртуальное окружение и ставят зависимости при первом запуске.

### Вручную (любая ОС)
```bash
python -m venv .venv
# Windows: .venv\Scripts\activate    |    macOS/Linux: source .venv/bin/activate
pip install -r requirements.txt
python -m presenter_overlay
```

> При **первом** включении микрофона `faster-whisper` скачает модель Whisper
> (для `base` ≈ 140 МБ). Дальше она кэшируется и работает офлайн.

## Как пользоваться
1. На экране появляется круглая кнопка — клик открывает/закрывает панель,
   правый клик — меню (выход).
2. Кнопка **«Микрофон»** — старт/стоп распознавания. Распознанные слова сразу
   попадают в поле ввода.
3. **➤** или текст в поле → отправка в DeepSeek. Ответ появляется в чате.
4. **⚙ Настройки**: API-ключ, выбор микрофона, язык и модель Whisper,
   модель ответа, контекст темы. «Сохранить» — параметры запоминаются в
   `~/.config/presenter-overlay/settings.json`.

Ключ OpenRouter уже встроен (`config.py → DEFAULT_API_KEY`), можно начинать сразу.
Клиент сам маршрутизирует: ключ `sk-or-…` → OpenRouter, иначе → нативный DeepSeek.

## Внешний вид и поведение окна
- **Стеклянный фон (glass, как в iOS).** На всех ОС панель рисует
  полупрозрачную стеклянную подложку (градиент + блик + тонкая рамка) ПОЗАДИ
  виджетов, поэтому кнопки и текст всегда видны. На Windows дополнительно
  включается реальное размытие за окном (DWM acrylic). На macOS используется
  нарисованное стекло (нативный слой размытия там ложился бы поверх контента и
  перекрывал кнопки, поэтому он отключён).
- **Всегда поверх всех окон и не пропадает при переключении приложений**
  (macOS: фоновая утилита без Dock-иконки, `hidesOnDeactivate=false`,
  floating-level + все Spaces/fullscreen; Windows: `HWND_TOPMOST`).
- **Свободное перетаскивание.** Тяни окно за шапку или любую пустую область —
  используется системное перетаскивание ОС. Кнопку-лаунчер тоже можно двигать
  (короткий клик по ней открывает/скрывает панель).

## Режимы ответа (без ручного «пинг-понга»)
В ⚙ Настройки → «Режим ответа»:

- **Текст: Whisper → DeepSeek** (по умолчанию). Локальный Whisper распознаёт речь,
  текст уходит в DeepSeek, который исправляет ошибки распознавания и отвечает.
  Галочка **«Авто-ответ»** убирает ручную кнопку: каждая завершённая фраза (после
  паузы) сама улетает в модель, и ответ появляется без нажатий.
- **Аудио: модель слушает звук напрямую.** Whisper не используется — звук фразы
  целиком отправляется аудио-модели, которая сама «слышит» и отвечает. Модель
  выбирается из списка в настройках (можно вписать свою). По умолчанию —
  `google/gemini-2.0-flash-001` (дёшево, ~$0.10/1M, аудио-вход через OpenRouter,
  тем же `sk-or-` ключом). Другие варианты: `google/gemini-2.5-flash`,
  `mistralai/voxtral-small-24b-2507`, `openai/gpt-audio`. Важно: **DeepSeek звук
  не принимает** — для этого режима нужна именно аудио-модель.

> Замечание: «авто-ответ» и «аудио-режим» применяются после нажатия «Сохранить»
> (прослушивание перезапустится автоматически).

## Скриншот → в нейросеть
Кнопка **📷** в панели (или назначенный хоткей «Скриншот → модель») делает
снимок экрана и отправляет его в **vision-модель**. Оверлей на момент снимка
прячется, чтобы не попасть в кадр; ответ появляется в чате.

В ⚙ Настройки:
- **Vision-модель** — модель, принимающая изображения (по умолчанию
  `google/gemini-2.5-flash`, дёшево; также `openai/gpt-4o` и др.).
- **«Что делать со скриншотом»** — твоя инструкция модели: найти ошибки,
  допинать код/текст на экране, объяснить и т.п.
- **«Выделить область»** — заранее выбери прямоугольник: снимок будет только
  этой зоны. На экране останется тонкая голубая рамка «📷 область», чтобы видеть,
  что попадёт в кадр (она click-through и не мешает работе, в сам снимок не
  попадает). Кнопка **«Весь экран»** сбрасывает область.

> macOS: для снимка экрана дай приложению/терминалу разрешение в **Системные
> настройки → Конфиденциальность и безопасность → Запись экрана**.

## Горячие клавиши (глобальные)
В ⚙ Настройки → «Горячие клавиши» можно назначить сочетания на **включение/
выключение микрофона** и **отправку запроса**. Они работают, даже когда окно не
в фокусе. Кликни в поле и нажми желаемое сочетание (например, ⌃⌥M); крестик
рядом очищает привязку.

> macOS: для глобальных хоткеев дай доступ приложению/терминалу в
> **Системные настройки → Конфиденциальность и безопасность → Универсальный
> доступ (Accessibility)**, иначе нажатия не будут приходить.

В аудио-режиме после ручной отправки (➤ или хоткей) микрофон по умолчанию
**выключается**, чтобы не слать лишние запросы (отключается галочкой в настройках).

## Захват системного звука (речь собеседника / видео)
По умолчанию приложение слушает **микрофон**. Чтобы распознавать **системный
звук** (голос собеседника в звонке, звук из видео), нужно «завести» системный
звук обратно во вход через loopback-устройство, а затем выбрать его в
⚙ Настройки → Микрофон. Такие устройства в списке помечены значком 🔊.

- **macOS (важно — одной установки BlackHole мало!):**
  BlackHole — это «труба», но звук в неё надо ещё направить. Просто поставить
  драйвер недостаточно: если вывод системы остаётся на динамиках, в BlackHole
  тишина, и приложение ничего не слышит. По шагам:
  1. Установи [BlackHole 2ch](https://existential.audio/blackhole/).
  2. Открой **Audio MIDI Setup** (Настройка Audio-MIDI) →
     «＋» внизу слева → **Create Multi-Output Device**.
  3. В нём поставь галочки на **BlackHole 2ch** И на своих динамиках/наушниках
     (чтобы ты сам продолжал слышать звук).
  4. В **Системные настройки → Звук → Выход** выбери этот **Multi-Output Device**.
     Теперь весь системный звук идёт и в колонки, и в BlackHole.
  5. В приложении → ⚙ → Микрофон выбери вход **🔊 BlackHole 2ch** и жми
     «Сохранить».
  6. Включи микрофон. Запусти видео — индикатор уровня в панели должен задёргаться
     (проценты рядом с «Слушаю»). Если 0% — звук не направлен в BlackHole
     (вернись к шагам 2–4).
- **Windows:** включи **«Стерео микшер» (Stereo Mix)** в «Звук → Запись» или
  поставь [VB-CABLE](https://vb-audio.com/Cable/). Выбери его в списке (🔊).
- **Linux (PulseAudio/PipeWire):** источник **«Monitor of …»** уже доступен как
  вход — просто выбери его в списке (🔊).

Распознанные слова появляются в поле ввода в реальном времени; DeepSeek по
контексту исправляет ошибки распознавания.

## Скрытие от захвата экрана
| ОС | Механизм | Статус |
|----|----------|--------|
| Windows | `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)` | работает (Win 10 2004+) |
| macOS | `NSWindow.sharingType = .none` (через pyobjc) | работает |
| Linux | системного аналога нет (X11/Wayland) | окно будет видно в записи |

Это скрывает окно от **программного** захвата (запись, Zoom/Meet, скриншоты),
но **не** от физической камеры, снимающей монитор.

## Настройки распознавания (Whisper)
Модель выбирается в ⚙: `tiny` (быстро/грубо) … `large-v3` (точно/медленно).
По умолчанию `base` — хороший баланс для CPU. Параметры near-real-time
(интервал частичного распознавания, порог тишины) — в `transcriber.py`.

## Структура
- `presenter_overlay/app.py` — точка входа.
- `presenter_overlay/overlay.py` — GUI: панель чата + кнопка-лаунчер (PySide6).
- `presenter_overlay/state.py` — состояние, связь распознавания/LLM/UI.
- `presenter_overlay/transcriber.py` — Whisper near-real-time + фиксация на паузах.
- `presenter_overlay/audio.py` — захват микрофона (sounddevice).
- `presenter_overlay/llm.py` — клиент DeepSeek/OpenRouter + промпт-суфлёр.
- `presenter_overlay/screen_privacy.py` — скрытие окна от захвата (платформенно).
- `presenter_overlay/config.py` — настройки (JSON в ~/.config).

## Если что-то не работает
- **Нет звука/микрофона** на Linux: поставь PortAudio — `sudo apt install libportaudio2`.
- **HTTP 401 / 402** — ключ недействителен или нет средств на OpenRouter.
- **Долгий первый запуск микрофона** — скачивается модель Whisper, это разово.
- **Тормозит распознавание** — выбери модель Whisper поменьше (`tiny`/`base`).
- **Пишет «Спокойная музыка» и подобный мусор** — это галлюцинации Whisper на
  музыке/тишине. Они фильтруются (Silero VAD + отсев по уверенности + чёрный
  список) в `transcriber.py`; пороги `NO_SPEECH_MAX`, `LOGPROB_MIN` и список
  `HALLUCINATION_PHRASES` можно подстроить под свой звук.

> Безопасность: ключ лежит в исходниках и в `~/.config`. Не публикуй проект с этим
> ключом. Для продакшена вынеси ключ в переменную окружения.
